## P0 — 프로덕션 투입 전 필수 (지금 착수)

### 1. OCR 원문 정제 (원문 품질 11% → 목표 90%+)
- [ ] PDF 텍스트 추출 단계에서 단어 중간 공백("미지 급", "공사 대금") 제거 후처리 규칙 추가
- [ ] 정제 대상 96건 재생성 — `input` 오염이 `output`으로 전이되지 않도록 재추출부터 수행
- [ ] 로더 개선 위치: `src/loaders.py` (PDF 추출) → `src/pipeline.py` STEP1 진입 전 정규화 훅
- [ ] 검증: 정제 후 "한글 1글자 고립 공백" 패턴 잔존율 5% 미만 확인

### 2. 질문 절단 수정 (질문 자연스러움 85% → 목표 98%+)
- [ ] 문서 제목을 고정 길이로 자르고 조사를 붙이는 로직 제거 ("대표발을(를)", "대여대에" 등 16건)
- [ ] 질문 생성 시 제목·문장 경계(어절 단위) 존중, 괄호 미완결 방지
- [ ] 재검수: `(` 개수 = `)` 개수, 조사 앞 단어 완결성 100%

### 3. 사실 근거성 검증 흔적 저장 (근거성 미검증 → 검증 가능화)
- [ ] STEP7 LLM Judge의 근거 점수/판정을 레코드 메타데이터에 저장 (현재 데이터에 흔적 없음)
- [ ] 각 응답에 `source_span`(원문 근거 구간) 필드 추가 → RAG·감사 추적성 확보
- [ ] 법률 도메인 특성상 오답 비용 큼 → 근거 미확인 레코드는 학습셋에서 제외 또는 플래그
- [ ] 관련 코드: `src/validate.py`, 직렬화 `src/export.py`
