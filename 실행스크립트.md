# 실행 스크립트

이 머신에는 시스템 파이썬이 여러 개 있어 의존성이 서로 다르다. 반드시 **프로젝트 전용
`.venv`의 파이썬**으로 실행한다. 맨몸 `python`/`uvicorn`을 쓰면 의존성 없는 파이썬이 잡혀
업로드 시점에 `No module named 'olefile'`(HWP) 같은 오류가 난다. (근거: CLAUDE.md)

```bash
# 0) 최초 1회 — 가상환경 생성 + 의존성 설치
python -m venv .venv
.venv/Scripts/python.exe -m pip install -r requirements.txt   # Windows
# .venv/bin/python -m pip install -r requirements.txt         # macOS/Linux

# 1) 파이프라인 1회 실행 (CLI)
.venv/Scripts/python.exe -c "from src.runner import run; run('samples/sample_admin.txt')"

# 2) 웹 UI 실행 → http://localhost:8000
.venv/Scripts/python.exe -m uvicorn web.app:app --reload

# 3) 검증 매트릭스(tests.md)
.venv/Scripts/python.exe -m pytest tests/ -q
```

## 선택 기능 (환경 변수로 켠다)

```bash
# 실제 Ollama 강제(기본 auto — 미가용 시 mock 폴백)
USE_MOCK_LLM=false .venv/Scripts/python.exe -m uvicorn web.app:app --reload

# 의미 유사도 + gold 기준 측정 (임베딩 모델 필요, docs/gold_reference.md 참고)
GOLD_PATH=gold SEMANTIC_ENABLED=1 .venv/Scripts/python.exe -m uvicorn web.app:app --reload

# 검증된 산출물 → gold 후보 추출(큐레이션). 사람 검토 후 gold/로 승격한다.
.venv/Scripts/python.exe -c "from src.gold import build_candidate_file as b; print(b('output/금융_20260715-104054/금융_dataset.json', 'gold/금융_후보.json'))"
```
