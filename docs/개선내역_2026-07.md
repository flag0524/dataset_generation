## 1. 생성 속도·오염 제거

### 근본 원인
1차 검증보고서가 지적한 "전체의 99.5%가 실제 생성이 아닌 템플릿 조립"의 원인은, 30초 SLA를 지키려 도입한 시간예산이 초과 청크를 휴리스틱 템플릿으로 폴백한 것이었다.

### 조치
- **폴백 제거** — 실제 LLM이 못 만든 앵글은 드롭(패딩 금지). mock 모드만 결정론적 구조 출력 유지(테스트용).
- **시간예산 기본 OFF** — 데이터셋 생성은 무제한(품질 우선). 웹 preview 모드에서만 옵인.
- **합성증강 제거** — min_rows 패딩 폐지. 크기 미달은 FAIL이 아니라 경고.
- **JSONL 정제본 생성** — Unsloth JSONL을 정제 후 최종 레코드에서 생성(JSON/CSV와 일치).
- **Alpaca 키 소문자** — `instruction/input/output`(Unsloth/HF 표준).

## 2. 키워드 품질

- 조사·접속사 **불용어** 제거.
- **후행 조사 분리**(`_strip_josa`) — 어간 2자 이상일 때만 분리해 '공제조합의→공제조합' 병합, '정의·국가'는 보존.
- **법령 인용 스캐폴딩 제거** — '제57조의3' 같은 인용의 중간 숫자가 한글 토큰을 쪼개 '조의·조제'를 만드는 것을 원천 차단.

## 3. 질문·대화형·키워드 고도화 (2차 보고서 대응)

- **질문** — `seg[:30]` 절단 폐기, LLM이 앵글별 자연어 질문 생성(괄호 미닫힘·`을(를)` 노출·청크번호 유출 해소).
- **대화형** — chatml/sharegpt를 `system=지시·user=원문·assistant=출력` 3턴으로 재설계(잘린 제목 암기 방지).
- **키워드** — 문서 단위 복사 → 청크별 LLM 키워드.
- **청크 게이트** — 최소 50자(`MIN_SEG_LEN`) + 신구조문 대비표 파편 필터.

## 4. 문서 로딩 견고화

- **ZIP 포맷(DOCX/XLSX/PPTX)** 비-ZIP 업로드 시 `BadZipFile`/`PackageNotFoundError`를 한국어 ValueError(400)로 변환.
- **실제 포맷 감지**(`_sniff_format`) — 매직 바이트·OLE 내부 스트림으로 구형 .xls/.doc/HWP/PDF/HTML/CSV를 식별해 안내.
- **손상 HWP** OLE 파싱 예외를 400으로.
- **웹 실행 인터프리터 고정** — 맨몸 `uvicorn` 대신 `python -m uvicorn`(의존성 설치 인터프리터 사용).

## 5. 이미지/스캔 PDF (OCR)

- **OCR 폴백** — pypdf 텍스트가 빈약한(<30자) 페이지만 pypdfium2로 렌더링해 Tesseract(kor+eng)로 채운다. 텍스트 PDF는 OCR 미탐(동작 불변).
- **OCR 정밀도** — 기본 PSM3(자동분할)이 한국어 문서에서 19%로 급락. **PSM6(균일 텍스트 블록)+300DPI**로 교체 → 실측 ~92%.
- **글자사이 공백 정규화** — Tesseract가 넣는 '건 설 산 업' 공백을 병합(3연속 이상만), 정상 단어 공백은 보존.
- 렌더러/Tesseract 미설치 시 조용히 빈 데이터 대신 명확한 ValueError.

## 6. 다중 소스·도메인 분류

- **다중 소스 통합**(`run_many`) — 여러 문서를 각각 생성 후 통합 재ID·중복제거·검증해 단일 산출물(법률_통합_*). 소스 1건 과적합 완화.
- **도메인 12종 확장** — 공공행정·법률·금융 + 외교·국방·교육·환경·노동·보건의료·산업경제·건설국토·농림수산.
- **LLM 2단 분류** — LLM 가용 시 문맥으로 주제 도메인 판정, 미가용/무효 응답이면 키워드 카운트 폴백. 예: 통일 안건 공공행정→외교 교정.

## 7. 신구조문대비표 실질 조문 보존

법안의 핵심인 신구조문대비표의 '개정안' 열(제14조의2 신설 등 실제 조문)이 '(현행과 같음)·<신 설>' 마커와 함께 통째로 드롭되던 문제 해결. 마커·대비표 채움·페이지조각만 제거하고 실질 내용이 남으면 청크로 살린다. 실측(2024598) 레코드 32→44.

## 8. DocumentAI 검증방법론 반영 (공공기관 기준)

방법론의 8단계·정량 기준을 검증 파이프라인에 심었다. 모두 망분리 대응(모델·LLM 사용분은 옵인·표본·오프라인 반입).

| 방법론 기준 | 파이프라인 | 실측(2024593/2024598) |
|---|---|---|
| Grounding 0.80+ | 엔티티 근거성(조문·금액·날짜·법령명·기관명, 문서 단위 대조) | 1.0 |
| Hallucination 2% 이하 | 원문에 없는 output 엔티티 플래그 | 0.0% |
| 의미 보존 0.95 | ko-sroberta 임베딩 코사인(옵인) | 0.824 |
| 중복 3% 이하 | `duplicate_rate` | 0.0% |
| Metadata 100% | `metadata_complete` | 100% |
| 자동평가(RAGAS) | 로컬 LLM 심판 faithfulness·answer_relevancy(옵인) | 0.713 / 0.938 |
| Human Review 5~10% | 위험도 우선 표본 → 검수 CSV | 2건/24 |
| 최종 90점 A | `grade` A/B/C/D | A |

### 문서 단위 환각 판정
엔티티 대조를 청크가 아니라 '같은 소스 문서 전체(그 문서 청크 합집합)'로 수행해, 다른 청크에 있는 조문 참조를 환각으로 오탐하던 것 제거(환각율 4.5%→0.0%).

### 한국어 임베딩
의미유사도 모델을 다국어 MiniLM → 한국어 특화 `jhgan/ko-sroberta-multitask`로 교체(판별폭 개선).
