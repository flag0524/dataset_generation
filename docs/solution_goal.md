## 1. 비전

망분리된 기관 내부망에서 실무자가 업무 문서를 올리면, 수작업 없이 검증된 AI 학습 데이터셋을 생성하는 안전한 노코드 플랫폼.

## 2. 핵심 목표 (기획서 §3.2)

1. 8단계 파이프라인 제품화 — 문서 → 데이터셋 자동화율 90% 이상.
2. 전 과정 온프레미스·오프라인 동작(외부 통신 0건).
3. Instruction·Q&A·RAG 3종 + Unsloth 학습 포맷 4종(Raw/Alpaca/ShareGPT/ChatML) 출력.
4. 검증 우선 루프(Validate → Preview → Refine → Run)로 품질 점수 90점 이상, PASS 95% 이상.
5. CPU 또는 소형 GPU 환경에서 동작하는 경량 구성.

## 3. 이번 사이클 실행 범위 (Now/MVP)

기획서는 M1~M6 로드맵이지만, 본 사이클에서 **코드로 구현**하는 범위는 아래로 한정한다(의사결정 확정). 보안·인증(JWT/SSO)·망분리 패키징은 인프라 결정 의존이라 별도 사이클로 분리한다.

## 4. 성공 기준 (Definition of Done, 본 사이클)

- `solution_*.md` 4종이 docs/에 존재하고 서로 일관된 계획·검증 구조를 이룬다.
- 산출물 파일명이 도메인 한글명 접두를 따른다(예: `공공행정_dataset.csv`). 웹 다운로드·대시보드 링크·테스트가 새 규칙과 일치한다.
- 검증 게이트 임계값(min_rows, quality_pass_score)이 환경변수/설정으로 조정 가능하다.
- runner가 단계별 진행률 이벤트를 외부로 전달할 수 있다.
- HWP/OCR 로더가 결정된 파서로 동작하거나, 미가용 시 명확한 한계 메시지로 폴백한다.
- `python -m pytest tests/ -q` 전부 통과.
- solution_status.md가 최종 상태로 갱신되고 변경이 커밋된다.

## 5. 비목표 (Non-Goals)

- 모델 파인튜닝(학습) 실행 — 외부(Unsloth)에 위임.
- 본 사이클에서의 보안·인증·망분리 패키징 구현.
- 대규모 멀티 GPU 학습 인프라.
